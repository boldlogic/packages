# httpserver

`httpserver` — небольшая обёртка над `net/http.Server` с типизированной конфигурацией, дефолтами и проверкой значений.

## Что делает пакет

- описывает конфигурацию HTTP-сервера через `ServerConfig`
- хранит сетевой адрес в полях `listen_ip` и `listen_port`
- хранит внешний хост в поле `external_host`
- задаёт таймауты сервера через `ServerOpts`
- создаёт `http.Server` с переданным `http.Handler`
- запускает сервер через `ListenAndServe`
- завершает сервер через `Shutdown` с контекстом вызывающего кода
- заполняет значения по умолчанию через `ApplyDefaults`
- проверяет диапазоны портов и таймаутов через `Validate`

## Что важно знать

- `ApplyDefaults` задаёт `127.0.0.1` для `listen_ip`, если поле пустое или состоит из пробелов
- `ApplyDefaults` задаёт `localhost` для `external_host`, если поле пустое или состоит из пробелов
- `ApplyDefaults` задаёт `8080` для `listen_port`, если порт равен `0`
- `ApplyDefaults` задаёт `20` секунд для `shutdown_timeout`, если значение равно `0`
- `Validate` считает корректными порты от `1` до `65535`
- `Validate` запрещает отрицательные значения `read_timeout`, `read_header_timeout`, `write_timeout` и `idle_timeout`
- `Validate` считает корректным `shutdown_timeout` от `1` до `60` секунд
- `NewServer` использует значения конфигурации как переданы вызывающим кодом; если нужны дефолты, вызовите `ApplyDefaults` до создания сервера
- `ShutdownTimeout` хранится в конфигурации для вызывающего кода, который создаёт контекст завершения сервера
