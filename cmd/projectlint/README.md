# projectlint

`projectlint` — служебная команда репозитория для проверок, которые не покрываются стандартными Go-инструментами.

Команда запускается из корня репозитория и анализирует файлы текущего дерева.

## Что проверяет команда

- комментарии к экспортируемым Go-типам, функциям и методам
- русскоязычность текстов ошибок и логов, если строка содержит буквы
- формат имён под-тестов `t.Run`
- соответствие версии `Текущий релиз` в корневом `README.md` локальным git-тегам формата `vX.Y.Z`

## Запуск

```bash
go run ./cmd/projectlint
```

## Правила GoDoc

Для экспортируемых типов, функций и методов комментарий должен начинаться с имени символа.

```go
// DecodeConfig декодирует файл конфигурации в структуру типа T.
func DecodeConfig[T any](path string) (T, error)
```

## Правила t.Run

Имя под-теста должно состоять из сегментов через `_`. В сегментах допустимы кириллица, латиница, цифры и символы `-`, `:`, `/`, `;`, `=`.

```go
t.Run("успешный_разбор", func(t *testing.T) {
	// ...
})
```
