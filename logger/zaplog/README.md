# zaplog

`zaplog` — компактный пакет логирования на базе [`go.uber.org/zap`](https://pkg.go.dev/go.uber.org/zap). Он даёт небольшой и явный контракт конфигурации для сервисов, которым нужен структурированный логгер без разрастания собственного слоя инициализации вокруг `zap`.

## Что делает пакет

- создаёт готовый `*zap.Logger` из небольшого объекта конфигурации
- поддерживает выбор уровня логирования через уровни `zap`
- переключает формат вывода между console и JSON
- пишет либо в `stdout`, либо в один целевой файл
- при невозможности открыть файл автоматически возвращается к `stdout`
- добавляет информацию о месте вызова в каждый логгер, созданный через `New`

## Публичный API

```go
type Config struct {
    Level      string
    Format     string
    OutputFile string
}

func (c Config) Validate() []error
func New(cfg Config) *zap.Logger
```

## Конфигурация

### `Level`

Поле `Level` разбирается через `zapcore.ParseLevel`, поэтому пакет использует нативную модель уровней `zap`.

Значения, которые явно покрыты тестами:

- `debug`
- `info`
- `warn`
- `error`
- `panic`

Если значение не удаётся распарсить, `Validate()` возвращает ошибку.

### `Format`

- `JSON` включает JSON-encoder
- любое другое значение включает console-encoder

Сравнение для JSON-ветки выполняется без учёта регистра.

### `OutputFile`

- пустое значение: лог пишется в `stdout`
- корректный путь к файлу: лог дописывается в этот файл
- недоступный путь: пакет переключается на `stdout`

Файл открывается с флагами append, create и write.

## Пример конфигурации

```json
{
  "level": "debug",
  "format": "json",
  "output_file": "app.log"
}
```

## Пример использования

```go
package main

import (
    "log"

    zaplog "github.com/boldlogic/packages/logger/zaplog"
)

func main() {
    cfg := zaplog.Config{
        Level:      "debug",
        Format:     "json",
        OutputFile: "app.log",
    }

    if errs := cfg.Validate(); len(errs) > 0 {
        log.Fatalf("некорректная конфигурация логгера: %v", errs)
    }

    l := zaplog.New(cfg)
    defer l.Sync()

    l.Info("сервис запущен")
}
```

## Характеристики вывода

- логгер использует production encoder config из `zap`
- время форматируется как `2006-01-02 15:04:05.000`
- `New` возвращает логгер, созданный с `zap.AddCaller()`

## Особенности дизайна

Пакет намеренно остаётся узким по зоне ответственности:

- не занимается загрузкой конфигурации приложения
- не пишет одновременно и в файл, и в консоль
- не открывает расширенный набор настроек `zap` через собственный оборачивающий API

Такой подход помогает держать контракт компактным и предсказуемым там, где важнее устойчивый базовый логгер, чем большой набор режимов и опций.

## Проверка поведения

Текущие тесты покрывают:

- корректные и некорректные значения уровня логирования
- выбор writer для `stdout`, доступного файла и fallback при ошибке открытия файла
- создание логгера без паники в console- и JSON-режимах
